Root Cause

The `userData` object is null when the component attempts to render the activity list. This usually happens when the API call hasn't finished yet.

info

Explanation

You're calling `.map()` on an array that doesn't exist yet. React tries to render the component immediately, but your asynchronous data fetch is still in flight.

check_circle

Suggested Fix

// Use optional chaining
userData?.activities.map(...)

// Or a loading state
if (!userData) return <Loading />

Pro Tip

Use React Query or SWR to handle these loading states automatically. It prevents most "undefined" errors by design!
